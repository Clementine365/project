PSP

My Personal Schedule Planner

A simple, interactive planner to organize daily tasks, priorities, reminders, and notes.

Contents: Site Name | Site Purpose | Scenarios | Color Schema | Typography | Wireframe | Project Requirements | Implementation Plan

Site Name

Personal Schedule Planner (PSP)

Reason: This name is short, and can easly tell the users what the app is for.

Site Purpose

The purpose of this website is to provide a simple, helpful and interactive app for managing daily schedules. the users can:

  • Create and manage daily tasks.
  • Set reminders for important events
  • Record notes for quick reference
  • Review weekly schedules and progress

Scenarios

  • How can I keep track of my daily tasks?
    Users can add tasks to their planner, edit,and check them off when completed.
  • Can I view my weekly overview?
    A weekly planner page provides a calendar-like layout.
  • What if I need to save notes for later?
    A notes page allows saving and storing important notes using local storage.

Color Schema

Primary Color: #2a7de1 (blue) – used for headers and navigation.
Secondary Color: #f4f6f9 (light gray) – used as background.
Accent Color: #ff9800 (orange) – used for highlights, buttons, and alerts.

Typography

Headings: Poppins (sans-serif, bold)
Body Text: Roboto (sans-serif, regular)

Wireframe

Home Page Layout:

  • Header with logo and site name (psp)
  • Navigation bar
  • Main section with intro text and quick links
  • Footer with contact and copyright

Secondary Pages:

  • Week Planner – the app displays weekly view with tasks
  • Notes Page – simple text input and saved notes list

Project Requirements

This project will meet the following functional and developmental specifications:

  • Hosted on GitHub Pages
  • At least three pages with consistent navigation
  • Accessible, visually appealing, responsive design
  • Valid HTML and CSS
  • Optimized images with lazy loading
  • HTML form implemented
  • Dynamic features via JavaScript:
    • DOM interaction (select, modify, events)
    • Conditional branching
    • Objects, arrays, and methods
    • Template literals for output
    • Use of localStorage
  • No grammar or spelling issues

Implementation Plan

  1. Week 4: Create basic HTML structure and CSS styles, Add navigation and responsive layout and Implement form and JavaScript interactivity
  2. Week 5:Connect localStorage and test dynamic features, Implement form and JavaScript interactivity, Optimize, and validate.
  3. Week 6:Final, publish on GitHub Pages.
